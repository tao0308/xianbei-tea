## 一、项目描述：使用 Agent/AI 驱动构建的具体成果

### 项目名称
**仙贝奶茶店智能管理系统** — 一套完整的奶茶店数字化运营平台

### 1. 项目解决的核心痛点

传统中小型奶茶店面临三大数字化难题：

**① 管理断层：** 店长与员工之间缺乏实时协同工具。新品上架、库存预警、配料管理等依赖口头沟通或纸质表格，出错率高、响应慢。

**② 顾客体验不闭环：** 顾客到店点单无数字化入口，无法浏览菜单、选择加料、查看套餐，更无法保存地址和订单历史。外卖配送缺少标准化费用管理和地址簿功能。

**③ 研发效率瓶颈：** 从 Idea 到上线周期长，传统开发需要前端、后端、UI 设计师协同，中小店家无力承担。

**本项目的解决方案：** 通过 Claude Agent 驱动的**全栈 AI 协同开发**，一人即可完成从需求分析到前后端代码落地的完整闭环，在数小时内构建出包含 20+ 页面的完整管理系统和手机端点单应用。

### 2. 核心逻辑流

本项目的开发过程体现了**长链推理**与**多 Agent 协作**的典型模式：

```
用户需求（自然语言）
    ↓
[binary-thinking Skill] 头脑风暴 → 设计方案 → 用户确认
    ↓
[writing-plans Skill] 制定实施计划（任务分解、文件清单、接口定义）
    ↓
[subagent-driven-development SKill] 多子代理并行执行
    ├→ 子Agent #1: 后端实体 + Repository + Controller 创建
    ├→ 子Agent #2: 管理后台 Vue 页面构建
    └→ 子Agent #3: 顾客手机端 Vant UI 页面构建
    ↓
[systematic-debugging Skill] 系统性调试（错误追踪 → 根因分析 → 修复 → 验证）
    ↓
[UI-styling Skill] 精调设计系统（阴影/圆角/间距/过渡动画）
    ↓
[schedule Skill] 自动化任务（库存预警通知）
    ↓
功能验证 → 用户反馈 → 迭代优化
```

**长链推理体现在：**
- 库存扣减联动的完整数据流追溯（顾客选加料 → 下单 → 按名称匹配原料 → 自动扣减库存）
- 跨端数据同步（后台设置轮播图 → 手机端实时显示）
- 套餐定价与加料价格的级联计算逻辑
- 配送费在订单总价中的多层计算链路

**多 Agent 协作体现在：**
- 每一次任务分拆为 2-3 个子 Agent 并行执行（前后端同步开发）
- Agent 之间通过任务文件共享状态，主 Agent 负责最终集成验证
- 遇到 Bug 时自动调用 `systematic-debugging` Skill 进行系统性排查，而非盲目修改
